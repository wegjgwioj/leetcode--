```python
class Node:
    __slots__ = 'prev','next','key','value'
    def __init__(self,key =0,value=0):
        self.key =key
        self.value=value


class LRUCache:

    def __init__(self, capacity: int):
        self.capacity = capacity
        self.dummy=Node() #
        self.dummy.prev = self.dummy
        self.dummy.next = self.dummy
        self.key_to_node={} #


    def get_node(self,key:int)->Optional[Node]:
        if key not in self.key_to_node:
            return None
        node = self.key_to_node[key] 
        self.remove(node)
        self.push_front(node)
        return node
    
    def remove(self,x:Node)->None:
        x.prev.next = x.next
        x.next.prev=x.prev

    def push_front(self,x:Node)->None:
        x.prev =self.dummy
        x.next =self.dummy.next
        x.prev.next = x
        x.next.prev =x


    def get(self, key: int) -> int:
        node = self.get_node(key)
        return node.value if node else -1

    def put(self, key: int, value: int) -> None:
        node = self.get_node(key)  # get_node 会把对应节点移到链表头部
        if node:  # 有这本书
            node.value = value  # 更新 value
            return
        self.key_to_node[key] = node = Node(key, value)  # 新书
        self.push_front(node)  # 放到最上面
        if len(self.key_to_node) > self.capacity:  # 书太多了
            back_node = self.dummy.prev
            del self.key_to_node[back_node.key]
            self.remove(back_node)  # 去掉最后一本书
      
```

### 1. `__slots__`：

在 Python 中，默认情况下，每个对象（类的实例）都会有一个内部字典 `__dict__` 来存储它的属性。这使得 Python 非常灵活（你可以随时给对象添加新属性），但也有代价：**字典非常占内存**。

**`__slots__` 的作用：**
它告诉 Python：“这个类的对象**只允许**有这几个固定的属性，不要给它分配 `__dict__` 字典了。”


#### 为什么要在这个代码里用？

在 `LRUCache` 场景下，缓存可能会存成千上万个 `Node`。

* 如果不加 `__slots__`：每个 Node 都要背一个字典，内存占用巨大。
* 加了 `__slots__`：每个 Node 仅占用很少的固定内存。**当数据量达到百万级时，能节省 40%~50% 的内存。**

### 2. `Optional[Node]`：类型提示 (Type Hinting)

这是 Python 3.5 引入的**类型提示**语法，用来告诉读代码的人（以及编辑器）这个变量可能是什么类型。它不会影响代码的运行，但能减少 Bug。

**`Optional[Node]` 的意思是：**

> 返回值可能是 **`Node` 类型**，也可能是 **`None`**。
