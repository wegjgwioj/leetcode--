# 爬楼梯
分类讨论：
- 选 1：剩 n - 1 个台阶，f(n - 1) 种方法。
- 选 2：剩 n - 2 个台阶，f(n - 2) 种方法。

互相独立  **“加法原理”**  dfs[n] = dfs[n - 1] + dfs[n - 2]
边界：dfs[0] = 1, dfs[1] = 1
dfs[2] = dfs[1] + dfs[0] = 2
...
```python
# 会超时的递归代码
class Solution:
    def climbStairs(self, n: int) -> int:
        def dfs(i: int) -> int:
            if i <= 1:  # 递归边界
                return 1
            return dfs(i - 1) + dfs(i - 2)
        return dfs(n)

```
这里面有大量重复计算，比如 dfs(2) 会被计算两次，dfs(3) 会被计算三次，dfs(4) 会被计算五次，等等。

# 记忆化搜索
## 记忆已有的结果
@cache装饰器会自动缓存函数的返回值，当函数被调用时，如果参数已经存在于缓存中，则直接返回缓存中的结果，而不需要重新计算。这种方式可以显著减少重复计算，提高效率。
```python
class Solution:
    def climbStairs(self, n: int) -> int:
        @cache
        def dfs(i: int) ->int:
            if i<=1:
                return 1
            return dfs(i-1)+dfs(i-2)
        return dfs(n)
```

## 空间优化
```python
class Solution:
    def climbStairs(self, n: int) -> int:
        f0 = f1 = 1
        for _ in range(2, n + 1):
            new_f = f1 + f0
            f0 = f1
            f1 = new_f
        return f1
```
